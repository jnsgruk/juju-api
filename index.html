    Juju Client API v1.0

    Scroll down for code samples, example requests and responses. Select a language for code samples from the tabs above or the mobile navigation menu.

    This specification represents early but ongoing design work for a next-generation, RESTful Juju client API. It should not be used as reference for any currently released version of Juju, and is unlikely to be implemented until some time in 2024.

    The source code for this page, and the OpenAPI specification can be found on Github. You can also find a progress tracker that indicates which of the prior facade calls have been covered by this design.

    The OpenAPI Spec can be entered directly into the Swagger Editor for a slightly different experience when exploring the API if preferable.

    In addition to the new design, there is a naive conversion of the existing Juju facade schema available. This was generated from the upstream schema.json and is used solely for the purposes of reference, and enables existing schemas to be easily reused during the design on the new API where appropriate.

    Returns the definitions of all clouds supported by the controller that the logged in user can see.

    Example responses

    200 Response

    {
  "clouds": []
}

    Responses

    Status Meaning Description Schema
    200 OK List of supported clouds. CloudList
    403 Forbidden Forbidden. Error
    500 Internal Server Error Internal server error. Error
    This operation does not require authentication
